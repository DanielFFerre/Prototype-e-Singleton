# Prototype-e-Singleton
Aluno: Daniel Felipe Ferreira

Este sistema utiliza o padrão Singleton na classe GerenciadorAlunos para garantir um Registro Acadêmico Central (SIGA) único e global, impedindo a duplicidade de dados. A classe Aluno aplica o padrão Prototype através do método clone(), permitindo a criação rápida de novos estudantes a partir de modelos de cursos pré-definidos, onde apenas os dados individuais são alterados após a cópia. Essa arquitetura assegura a integridade da base de dados e otimiza a memória ao evitar instanciacões manuais repetitivas.
